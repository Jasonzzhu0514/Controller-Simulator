# 控制器模拟器

NMPC/MPC 基于模型滚动预测优化；LQR/PID 是根据当前状态反馈的控制基线。

这个仓库里有两个无第三方依赖的演示，网页模拟器放在 `web/` 目录：

- `nmpc_demo.py`：命令行版，展示 NMPC 的核心循环。
- `web/index.html`：网页模拟器入口，用来对比 NMPC、MPC、LQR、PID。
- `web/style.css`：网页模拟器样式。
- `web/simulator.js`：网页模拟器交互、仿真和绘图逻辑。
- `start.sh`：Linux/macOS 启动入口。
- `start.bat` / `start.ps1`：Windows 启动入口。

```text
Controller-Simulator/
├── README.md
├── README.zh-CN.md
├── nmpc_demo.py
├── start.bat
├── start.ps1
├── start.sh
└── web/
    ├── index.html
    ├── simulator.js
    └── style.css
```

## 运行命令行 Demo

```bash
python3 nmpc_demo.py
```

命令行 demo 展示：

1. 读取当前状态。
2. 枚举未来控制序列。
3. 用模型预测未来。
4. 给每条候选轨迹计算 cost。
5. 只执行最优序列的第一步。
6. 下一步重新计算。

## 打开网页模拟器

Linux/macOS：

```bash
./start.sh
```

Windows：

```bat
start.bat
```

PowerShell：

```powershell
.\start.ps1
```

启动脚本会启动本地 HTTP 服务并自动打开浏览器。默认从 `8008` 端口开始寻找空闲端口；如果要指定起始端口：

```bash
PORT=8010 ./start.sh
```

Windows PowerShell：

```powershell
$env:PORT=8010; .\start.ps1
```

也可以手动访问终端输出的地址，例如：

```text
http://127.0.0.1:8008/index.html
```

## 控制器

网页里有四种模式：

- `NMPC`：用非线性阻力模型预测未来。
- `MPC`：用线性模型预测未来。
- `LQR`：线性二次型调节器风格的状态反馈。
- `PID`：普通 P/I/D 反馈控制基线。

页面右上角提供说明弹窗和语言切换，默认英文，可切换简体中文、繁体中文、日语和俄语。

三种模式控制的是同一个真实非线性对象：

```text
x_dot = v
v_dot = u - drag * v * |v|
```

因此可以直观看到：

- NMPC 的预测模型更接近真实对象。
- MPC 仍然预测未来，但线性模型忽略阻力。
- LQR 不预测约束，只按位置误差和速度做线性状态反馈。
- PID 不预测未来，只根据当前误差、积分误差和速度反馈。
- 在限速区场景里，MPC/NMPC 会提前把未来超速计入 cost；PID 只能到当前状态变化后再反应。
- NMPC/MPC 接近目标后会进入终端捕获/保持逻辑，避免粗离散控制档位在目标点附近来回周旋。

## 可调参数

场景参数：

- 目标位置
- 真实阻力系数
- 限速区起点
- 限速区速度上限
- 限速惩罚权重

MPC/NMPC 参数：

- 预测步数 horizon
- 控制候选档位
- 位置误差权重
- 终端误差权重
- 速度惩罚权重
- 控制平滑权重
- 控制分段数
- 控制能量权重
- 终端速度倍率
- 终端位置/速度增强
- 收敛半径和收敛速度权重
- 目标保持阈值
- 目标捕获范围、捕获增益和捕获限幅

PID 参数：

- 比例增益 `Kp`
- 积分增益 `Ki`
- 微分增益 `Kd`
- 积分限幅

LQR 参数：

- 位置反馈 `Kx`
- 速度反馈 `Kv`

按钮说明：

- 左上角 `Reset`：只重置小车状态和曲线。
- 右侧 `Reset Params`：按当前控制器恢复对应默认参数。
- 右侧 `Reset All`：同时恢复参数并重置小车状态。

当前默认预设：

- NMPC：horizon 28，候选档位 7，位置权重 1.2，终端权重 13，速度权重 0.34，平滑权重 0.34。
- MPC：horizon 32，候选档位 7，位置权重 0.9，终端权重 16，速度权重 0.58，平滑权重 0.42。

## 显示说明

页面布局已按桌面视口压缩，目标是不出现页面滚动条；如果浏览器窗口很矮，建议全屏查看。

顶部工具栏里的 `Speed` 支持慢放和快放：小于 `1x` 会隔帧执行仿真步，大于 `1x` 会每帧执行多个仿真步。

曲线图使用自适应坐标轴：

- y 轴会根据最近历史中的目标、位置、速度、控制量自动扩展范围。
- y 轴显示动态数值刻度。
- x 轴显示仿真时间 `t`，单位是秒。
- 曲线从 `t = 0` 开始保留，右边界随当前仿真时间不断延伸；显示时会抽样绘制长历史，数据本身不再按滑动窗口删除。
- 当系统已经稳定在目标点附近时，曲线会继续记录 5 秒稳定尾段，然后停止追加新点；此后控制流也会停在最后状态，不再循环播放。

页面底部还包含实时控制流可视化：

- NMPC/MPC：显示读取状态、生成候选、ForLoop 遍历、rollout、cost、best command、执行第一步。
- PID：显示误差、P 项、I 项、D 项、合成、限幅、输出。
- LQR：显示状态误差、Kx 反馈、Kv 反馈、合成、限幅、输出。
- 控制流使用蓝图式 SVG：白线表示执行顺序，绿线表示状态/候选数据，蓝线表示对象更新后的 `x,v` 反馈回下一帧输入。

切换控制器时，控制流节点会自动切换；运行时会按实际计算状态高亮路径并显示关键数值，稳定冻结后不再循环播放。
